2. Skill content (≈150 lines)

  • Frontmatter: name: netsuite-cli; description triggers on NetSuite REST data tasks (records, SuiteQL, schemas, RESTlets, async jobs) from any project — including phrases like "query NetSuite", "get/update a NetSuite record", "NetSuite schema".
  • Command chooser table: know type+id → record get · query/aggregate/join → suiteql · unknown shape → describe · unsupported endpoint → raw · long-running → job submit · deployed script → restlet call.
  • Bootstrap: --version check → install (GitHub release download or cargo install --git) → account list → account add (M2M for unattended, auth-code for named user; NetSuite-side setup lives in the README) → account test. Account selection precedence (--account flag → NETSUITE_ACCOUNT → default).
  • Recipes: describe-first discovery; --all pagination (honest hasMore:true on guard stops); create returns id via Location; upsert by eid:; async submit → status → result; SuiteQL count/aggregate with string-typed values; metadata-catalog select= single-object quirk.
  • Error triage: exit 3 → reauth per flow · exit 1 → parse o:errorCode/details · exit 2 → usage, re-read --help · exit 4 → network (retries already happened) · 429/5xx handled internally. Windows = EC P-256 keys only. Auth-code refresh rotation races under parallel invocations → use M2M for agent fleets.

3. README quick-start section

Inserted after the intro paragraph, before "Install": fenced install one-liner per OS (curl release asset → unzip → PATH, plus cargo install --git alternative), then the four bootstrap commands, then record get/suiteql/describe one-liners, then two pointers: "full command reference below" and "agents: load skills/netsuite-cli/SKILL.md". No duplication of the full reference.

4. Constraints & non-goals

  • No account IDs, cert IDs, client IDs, or internal URLs anywhere in repo artifacts (public repo — enforced lesson from PR #2).
  • Skill follows superpowers:writing-skills conventions (tight, imperative, no restating --help).
  • Every command example must parse against the real binary before merge (same bar as the README had).
  • Non-goals: no plugin packaging, no changes to CLI code, no AGENTS.md (the skill covers it), no rewrite of existing README sections.

5. Testing & delivery

  • Validate every skill/README command against target/release/netsuite-cli (parse-level; live calls only for read-only examples).
  • Symlink verified by listing skills in a fresh session context (manual check).
  • Branch feature/agent-skill → PR → pr-review-loop to green.
